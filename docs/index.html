Technical Architecture & Implementation Deep Dive

1. User-Space Forwarding Proxy Engine

The core of the Smart Hotspot Manager is a high-performance HTTP/HTTPS forwarding proxy server implemented using Dart's asynchronous TCP socket framework (`dart:io`). Traditional Android networking tools often require root access to manipulate `iptables` for traffic interception. Our solution bypasses this requirement by binding a `RawServerSocket` to the hotspot's gateway interface (standardized at `192.168.43.1` on port `8080`).

When a client device connects to the hotspot and configures its manual proxy, every outbound request is received by our Dart isolate. The system handles the entire lifecycle of the socket connection, from the initial SYN/ACK handshake to the final teardown.

2. HTTP CONNECT & Secure Tunneling

Handling encrypted HTTPS traffic without performing MITM (which would break security certificates) requires the implementation of the HTTP CONNECT method. When a client requests a secure site, it sends an unencrypted header: `CONNECT targethost.com:443 HTTP/1.1`. The Smart Hotspot Manager parses this plaintext header to identify the destination before any encryption begins.

Once the destination is validated against security rules, the app establishes a raw TCP connection to the remote server and pipes the byte stream symmetrically between the client and the server. This "blind piping" ensures that while the content remains private, the metadata (destination and throughput) is fully observable.

3. Layer-7 Deep Packet Inspection (DPI)

The firewall system utilizes a specialized form of Deep Packet Inspection. Unlike packet sniffers that analyze every fragment, our engine inspects the "Client Hello" or the initial HTTP "Host" header. By analyzing the Server Name Indication (SNI) in the TLS handshake, we can determine exactly which domain is being accessed even if the data itself is encrypted. If the domain exists in our `SecurityRulesManager` blacklist, the connection is pre-emptively terminated before the TLS handshake completes.

4. Token Bucket Throttling Mechanism

To enforce bandwidth limits (e.g., 1024 Kbps), the system implements a software-defined Token Bucket algorithm. We calculate the required delay for every chunk of data forwarded through the socket. If a chunk of `N` bytes is processed and the limit is `L` bytes/sec, the system introduces an artificial latency using `Future.delayed()`. This approach is superior to packet dropping as it allows TCP's congestion control algorithms to adjust naturally, providing a smoother user experience without wasting network resources.

5. Concurrency & Performance Optimization

  • Asynchronous Isolates: The networking logic runs on Dart's event loop, ensuring that I/O operations do not block the main UI thread.
  • Foreground Service Persistence: Using `flutter_foreground_task`, the proxy server is bound to a persistent Android Service. This prevents the OS from killing the networking process when the app is in the background or the device is in low-power mode.
  • Provider Pattern State Management: Traffic metrics (Bytes per second) are aggregated and debounced using the Provider pattern, ensuring that the real-time UI charts are updated efficiently without excessive rebuilds.
